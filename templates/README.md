# Bootstrap templates

Reference implementation for the bootstrap skill. **Adapt** into a new app **after** `bunx create-expo-app@latest … --template default` and removing default cruft.

**Non-interactive:** new folder → `bunx create-expo-app@latest <APP_NAME> …`. Existing `.git` → `CI=true bunx create-expo-app@latest . …`.

**Source:** https://github.com/hosam-hubspire/expo-project-bootstrap

## Adaptation workflow

1. **Compare** scaffolded files with templates — merge, do not bulk-copy `package.json`, `app.json`, `tsconfig.json`.
2. **Install** dependencies (grouped commands below) — skip groups for stack items the user unchecked at intake.
3. **Add** template files: lint config, `scripts/`, CI, `.rnstorybook/`, `codegen.ts`, `FIGMA_EXPORT.md`, `src/`, `assets/` (including `assets/icons/nanoicons.ttf` and `nanoicons.glyphmap.json`).
4. **Replace** demo routes with the app shell from `src/app/`.
5. **Strip down** when the user unchecked i18n, GraphQL, or Storybook — see [`optional/minimal/README.md`](./optional/minimal/README.md).
6. **Figma tokens** (when a design-system URL is provided): Phase B after `bun install` — [`FIGMA_EXPORT.md`](./FIGMA_EXPORT.md).
7. **Argent:** `bun install -g @swmansion/argent`; `bunx @swmansion/argent init -y` in project root.

## Installing dependencies

Package checklist only — no `templates/package.json`, no hand-written version ranges.

Install in **logical groups** with `--verbose`. After all applicable groups: `bun install --verbose`.

### Core (always)

```bash
# Expo / localization (i18n device locale)
bunx expo install --verbose expo-localization

# Icon font loading at runtime
bunx expo install --verbose expo-font

# Styling + state
bun add --verbose uniwind tailwindcss zustand react-native-mmkv react-native-nitro-modules

# Icons (nano-icons font pipeline)
bun add --verbose react-native-nano-icons

# Lint + format
bun add -d --verbose @biomejs/biome eslint eslint-plugin-react-native-a11y typescript-eslint

# Test
bun add -d --verbose jest jest-expo @testing-library/react-native @types/jest
```

### Default stack (install unless unchecked at intake)

```bash
# i18n
bun add --verbose i18next react-i18next

# GraphQL runtime
bun add --verbose @apollo/client graphql graphql-ws apollo3-cache-persist @graphql-typed-document-node/core

# GraphQL codegen (dev)
bun add -d --verbose @graphql-codegen/cli @graphql-codegen/client-preset

# Storybook (dev)
bun add -d --verbose storybook @storybook/react-native @storybook/addon-ondevice-actions @storybook/addon-ondevice-backgrounds @storybook/addon-ondevice-controls @storybook/addon-ondevice-notes

bun install --verbose
```

After applying templates with GraphQL enabled, set `EXPO_PUBLIC_GRAPHQL_URL` and run `bun run graphql:generate` when `.graphql` operations change — templates ship committed minimal output under `src/services/graphql/generated/`.

### Project fonts (after Figma export)

Install `@expo-google-fonts/*` or custom font packages matching **your** Figma `fontFamily` values. Load them with `expo-font` (extend `IconFontLoader` or add a dedicated font loader) and align native font keys in `src/theme/fonts.ts`. Typography classes use **exported family names** via CSS variables in `src/theme/tokens/generated/font-families.css` (regenerated by `bun run tokens:generate`) — not Tailwind `font-sans` / `font-mono` slots. Until Phase B, stub tokens use system stacks.

Set **`EXPO_PUBLIC_GRAPHQL_URL`** to your project GraphQL HTTP endpoint before running the app or `bun run graphql:generate`. Templates ship a minimal `ExampleQuery` placeholder in `src/services/graphql/operations/`.

GraphQL **subscriptions** (when GraphQL is enabled): set `EXPO_PUBLIC_GRAPHQL_SUBSCRIPTIONS_ENABLED=true` and optionally `EXPO_PUBLIC_GRAPHQL_WS_URL`.

## Scripts

Merge into scaffolded `package.json`:

| Script | Command | When |
|--------|---------|------|
| `lint` | `biome check . && eslint .` | always |
| `lint:fix` | `biome check --write . && eslint . --fix` | always |
| `lint:a11y` | `eslint .` | always |
| `test` | `jest --ci` | always |
| `test:watch` | `jest --watchAll` | always |
| `tokens:generate` | `node scripts/generate-design-tokens.mjs` | always |
| `graphql:generate` | `graphql-codegen --config codegen.ts` | GraphQL enabled |
| `storybook` | `EXPO_PUBLIC_STORYBOOK_ENABLED=true bunx expo start` | Storybook enabled |
| `storybook-generate` | `sb-rn-get-stories` | Storybook enabled |

## Icons

`react-native-nano-icons` in `app.json`: `inputDir` and `outputDir` → `./assets/icons`. Sample SVGs and matching `nanoicons.glyphmap.json` / `nanoicons.ttf` ship in templates.

Export new SVGs from Figma into `assets/icons/`, then `bunx expo prebuild` to regenerate the font.

## Argent

Run `bunx @swmansion/argent init -y` in the project root during bootstrap.

## Raw token stubs

`src/theme/tokens/raw/` ships **placeholder sample files** for CI before Phase B — replace entirely during export. Stub typography uses a generic `"System"` family name; Phase B replaces it with your Figma families. See [`FIGMA_EXPORT.md`](./FIGMA_EXPORT.md).
